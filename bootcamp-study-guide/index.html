HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Every element starts with a tag and closes with the same tag.
  • After you save the code, you will be asked to commit it for it to be shown in the main code.
  • Every tag that you create will automatically create the closing tag.

CSS

  • CSS serves as a style giver for HTML.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • .card serves as a style for just a selection of elements.
  • External style is the most flexible way to style a HTML code because you separate both codes. It's also the most organized way.

Git

  • Every command for git should be made form the Terminal.
  • git status: checks what branch we are currently on.
  • git checkout -b "branch-name": creates a new branch and switches to it.
  • git add -A: Used to commit all files in the working branch.
  • git commit -m: Used to commit changes into out repo.
  • I think I still need to link my email with git for commits.
  • git pull origin main: Makes the branch up to date before making the PR.
  • git push origin "branch name": Makes the push to the repo in GitHub.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Following the flow structure, first you need to declare de core variables and then you can create and call functions.
  • We use "console.log("TEXT"); to print text directly to the console.
